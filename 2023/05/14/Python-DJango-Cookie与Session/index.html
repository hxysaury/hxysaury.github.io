Cookie与Session

HTTP被设计为”==⽆态==”，也就是俗称“脸盲”。 这⼀次请求和下⼀次请求 之间没有任何状态保持，我们⽆法根据请求的任何⽅⾯(IP地址，⽤户代理等)来识别来自同⼀ 个⼈的连续请求。

实现状态保持的⽅式：在客户端或服务器端存储与会话有关的数据 （客户端与服务器端的⼀次通信，就是⼀次会话）

  • cookie
  • session

不同的请求者之间不会共享这些数据，cookie和session与请求者⼀⼀对应
